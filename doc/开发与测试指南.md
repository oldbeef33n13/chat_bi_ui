# 开发与测试指南

这份文档单独说明三件事：

- 如何一键跑完整测试
- 如何做 `web` 前端本地开发
- 如何做 `chatbi-app-server` 后端本地开发

适用场景：

- 新机器初始化
- 本地联调
- 功能调试
- showcase 演示
- 端到端自动化验证

## 1. 前置环境

需要先安装：

- Node.js 20+
- npm 10+
- Python 3.11+
- JDK 21
- Maven 3.9+

默认端口：

- 前端 dev：`5173`
- 前端 showcase：`4173`
- 后端 API：`18080`
- AI 服务：`18180`

## 2. 一键测试

### 2.1 首次初始化

```bash
npm run stack:init
```

这个命令会完成：

- 安装前端依赖
- 编译后端测试类和依赖
- 初始化 Python AI 服务虚拟环境并安装依赖
- 安装 Playwright Chromium

### 2.2 一键跑完整链路

```bash
npm run test:all
```

它会按顺序执行：

1. 初始化依赖
2. 后端集成测试
3. 前端类型检查
4. Python AI 服务测试
5. 前端单测
6. 启动 showcase 栈
7. 执行 Playwright 全链路 smoke
8. 自动停止服务

这是当前推荐的主验证入口。

### 2.3 单项测试命令

前端类型检查：

```bash
npm run typecheck
```

前端单测：

```bash
npm run test
```

后端测试：

```bash
npm run test:backend
```

Python AI 服务测试：

```bash
.runtime/venv/chatbi-ai-service/Scripts/python -m pytest tools/chatbi-ai-service/tests
```

浏览器 e2e：

```bash
npm run test:e2e
```

说明：

- `test:e2e` 需要先启动可访问的前后端
- 默认访问 `http://127.0.0.1:4173`
- 如需指定地址，可以设置 `CHATBI_E2E_BASE_URL`

PowerShell 示例：

```powershell
$env:CHATBI_E2E_BASE_URL="http://127.0.0.1:4173"
npm run test:e2e
```

## 3. Web 前端开发

### 3.1 推荐方式：一键联调

最省事的方式：

```bash
npm run stack:dev
```

这个命令会同时启动：

- `chatbi-app-server`
- `chatbi-ai-service`
- Vite dev server

启动时会额外输出：

- 当前使用的 `.runtime/storage/dev` 目录
- 后端日志文件路径
- AI 服务日志文件路径
- 前端日志文件路径

如果启动失败，脚本会直接把后端/前端日志尾部打印到控制台。

如果检测到本地 `.runtime/storage/dev` 里的 SQLite/Flyway 校验和当前代码不兼容，脚本会自动重置这个目录并重试一次。

启动成功后访问：

- `http://127.0.0.1:5173/#/docs`

停止：

```bash
npm run stack:stop
```

### 3.2 手动启动前端

如果只想单独调前端：

```bash
npm install
npm run dev
```

Vite 默认会把：

- `/api`
- `/files`
- `/ai-api`

代理到：

- `http://localhost:18080`
- `http://localhost:18180`

对应配置在 [vite.config.ts](/d:/GitHub/chat_bi_ui/vite.config.ts)

如果后端不是默认端口，可以改代理目标：

```powershell
$env:VITE_API_TARGET="http://localhost:19080"
npm run dev
```

```powershell
$env:VITE_AI_TARGET="http://localhost:18180"
npm run dev
```

### 3.3 前端开发时的用例准备

前端当前不再走本地 mock REST，默认依赖真实 Java 后端。

也就是说前端开发要有一份可用后端数据。推荐两种方式：

1. 直接用 `npm run stack:dev`
2. 先手动启动后端，再单独跑 `npm run dev`

后端使用空存储目录启动时，会自动准备种子模板和内置数据接口，所以通常不需要手工造数。

如果你想重置联调用例：

```powershell
Remove-Item .runtime\storage\dev -Recurse -Force
npm run stack:dev
```

### 3.4 前端开发时常用验证

改完前端后，建议至少执行：

```bash
npm run typecheck
npm run test
```

如果改动影响到真实页面流程，再补：

```bash
npm run stack:showcase
npm run test:e2e
```

## 4. 后端开发

### 4.1 手动启动 Java 服务

推荐命令：

```bash
mvn -f tools/pom.xml -pl chatbi-app-server -am org.springframework.boot:spring-boot-maven-plugin:run
```

如果要显式指定本地存储目录，推荐这样起：

```powershell
mvn -f tools/pom.xml -pl chatbi-app-server -am org.springframework.boot:spring-boot-maven-plugin:run "-Dspring-boot.run.jvmArguments=-Dapp.storage.base-dir=.runtime/storage/dev"
```

服务默认地址：

- `http://127.0.0.1:18080`

健康检查：

- `http://127.0.0.1:18080/api/v1/health`

### 4.2 后端开发时的用例准备

后端使用 SQLite，本地数据目录来自：

- 默认：`storage/`
- 一键脚本：`.runtime/storage/dev` 或 `.runtime/storage/showcase`

第一次启动空目录时，会自动完成：

- SQLite 建库
- Flyway migration
- 种子模板准备
- 内置运维数据接口准备

所以后端本地调试通常不需要额外造数。

如果要重置后端本地用例：

```powershell
Remove-Item .runtime\storage\dev -Recurse -Force
```

然后重新启动后端即可。

运行脚本后的运行时目录：

- 日志：`.runtime/logs/`
- 本地联调数据：`.runtime/storage/dev`
- showcase 数据：`.runtime/storage/showcase`

### 4.3 后端开发时常用验证

后端改动后，至少执行：

```bash
npm run test:backend
```

如果改动影响到模板预览、导出、调度或前端联调，再执行：

```bash
npm run test:all
```

## 5. Python AI 服务开发

### 5.1 手动启动 AI 服务

推荐命令：

```powershell
python -m venv .runtime/venv/chatbi-ai-service
.runtime/venv/chatbi-ai-service/Scripts/python -m pip install -e tools/chatbi-ai-service[dev]
.runtime/venv/chatbi-ai-service/Scripts/python -m uvicorn app.main:app --host 127.0.0.1 --port 18180 --app-dir tools/chatbi-ai-service
```

服务地址：

- `http://127.0.0.1:18180`

健康检查：

- `http://127.0.0.1:18180/health`

### 5.2 当前已承载的 AI 能力

- `command-plan`
- `chart-recommend`
- `chart-ask`
- `data-guide`
- `story-summary`

当前默认是规则兜底模式，不依赖外部模型也能本地跑通。

如果需要接真实模型，推荐把配置放在：

- `.runtime/ai.env`
- 或 `tools/chatbi-ai-service/.env.local`

可参考：

- `tools/chatbi-ai-service/.env.example`

### 5.3 AI 服务开发时常用验证

```powershell
.runtime/venv/chatbi-ai-service/Scripts/python -m pytest tools/chatbi-ai-service/tests
```

## 6. Showcase 演示

如果你要跑完整展示环境：

```bash
npm run stack:showcase
```

访问地址：

- `http://127.0.0.1:4173/#/docs`

这个模式会：

- 重新 build 前端
- 使用干净的 showcase 存储目录
- 启动后端
- 启动 AI 服务
- 启动 `vite preview`

适合：

- 产品演示
- UI 截图
- 端到端 smoke

## 7. 日志与故障排查

开发态日志：

- `.runtime/logs/backend-dev.log`
- `.runtime/logs/ai-dev.log`
- `.runtime/logs/frontend-dev.log`

showcase 日志：

- `.runtime/logs/backend-showcase.log`
- `.runtime/logs/ai-showcase.log`
- `.runtime/logs/frontend-showcase.log`

常见问题处理：

1. 端口占用

```bash
npm run stack:stop
```

2. 想重新准备种子数据

```powershell
Remove-Item .runtime\storage\dev -Recurse -Force
Remove-Item .runtime\storage\showcase -Recurse -Force
```

3. Playwright 浏览器缺失

```bash
npm exec playwright install chromium
```

## 8. 推荐开发路径

日常前端开发：

1. `npm run stack:dev`
2. 改前端代码
3. `npm run typecheck`
4. `npm run test`

日常后端开发：

1. 手动起 `chatbi-app-server`
2. 前端用 `npm run dev` 联调
3. `npm run test:backend`

提交前：

1. `npm run test:all`
