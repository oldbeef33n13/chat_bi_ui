# 1. 背景与愿景

## 1.1 背景

面向 ChatBI 的网络运维场景（NOC/NetOps），用户需要在对话中完成：

* 查询数据 → 自动生成图表
* 组装为 Dashboard / Report / PPT
* 通过对话或可视编辑快速修改样式、布局、过滤条件与内容结构
* 最终交付为可视化大屏、周报/月报、汇报 PPT

现实中这类需求被拆散在多个工具里（图表工具/大屏工具/报告工具/PPT 工具），导致：

* 复用差、学习成本高、体验割裂
* AI 生成难以与编辑器协同（缺少命令系统、diff预览、撤销审计）
* 工程扩展困难（地图/拓扑/表格/富文本后续接入成本高）

## 1.2 愿景

构建一个“可视化内容构建操作系统（Visual Document OS）”：

* **统一 DSL + 统一编辑内核 + 插件化节点体系**
* 单图表为基础能力，Dashboard/Report/PPT 复用同一引擎
* ChatBI 通过“命令计划”驱动画布对象，实现**可控、可审计、可撤销**的 AI 编辑

# 2. 目标与成功标准

## 2.1 体验目标（必须）

1. **低门槛**：模板/片段/主题一键应用，默认就“好看能用”
2. **丝滑**：拖拽/缩放 <16ms/帧；大文档不掉帧；图表懒加载
3. **专业可控**：快捷层覆盖高频；属性面板覆盖全量配置
4. **AI可控**：对话修改先预览 diff，支持接受/撤销/审计
5. **一致性**：四类编辑器共享同一套选择、对齐、样式系统与快捷键

## 2.2 工程目标（必须）

* 统一 Document Model（Node Tree）与 Patch 机制
* 所有修改必须经过 Command（包括 UI 和 AI）
* 插件化：节点类型/渲染器/属性面板/命令可注册
* DSL 结构平铺：降低 LLM 生成复杂度、提升稳定性
* 主题 tokens 一体化：UI 与图表主题统一

## 2.3 非目标（阶段性不做）

* 多人实时协作（但提供版本快照/回滚）
* 高级富文本排版（保留扩展点）
* 超复杂矢量图形编辑（后续插件化接入）

# 3. 技术栈与约束（新增）

## 3.1 前端技术栈

* 框架：React 18+
* 状态管理：@preact/signals-react（Signals）
* 图表：Apache ECharts 5+
* 语言：TypeScript（strict）
* 样式：Less（自研 Design Tokens + Theme）
* i18n：自研轻量 i18n（或兼容 i18next，但 UI 组件库不强依赖第三方）
* 构建：Vite / Webpack（可选，建议 Vite）
* 单测：Vitest / Jest（可选）
* E2E：Playwright / Cypress（可选）

## 3.2 UI 组件库约束

* 不依赖第三方 UI 大组件库（Antd/MUI 等）
* 必须提供：

  * Drawer / Menu / Toolbar / Popover / ColorPicker / Dropdown / Tabs / Tree / List
  * 可访问性（基础）
  * tokens 主题能力（dark/light/brand）
  * i18n 文案切换
  * 可扩展样式插槽

## 3.3 性能与工程约束

* 拖拽期间只更新 transform（UI state），释放后提交 DSL patch
* ECharts 实例按 viewport 懒创建/销毁（或降级预览）
* Report/PPT 长列表虚拟化
* 网络请求：支持取消（AbortController）+ debounce + cache（SWR 可选）

# 4. 总体架构

## 4.1 分层架构

1. **Document Layer**：统一 DSL（VDoc/VNode），资产、数据源、过滤器
2. **Editor Kernel**：Selection / Command / Patch / History / Transaction
3. **Runtime Layer**：Renderer Registry + Layout Engine + Data Engine
4. **UI Layer**：Toolbar / Inspector / Panels / Canvas / Tree / Thumbnails
5. **Chat Bridge**：命令计划生成、diff 预览、执行与回滚、上下文理解

## 4.2 关键原则

* UI 不直接改 Document，**必须通过 Command**
* AI 修改与 UI 修改等价：**同一命令体系、同一撤销体系**
* DSL 平铺设计：绑定表/开关/样式尽量平铺，降低嵌套

# 5. 统一 DSL 设计（VDoc/VNode）

## 5.1 顶层 Document

```ts
type DocType = "chart" | "dashboard" | "report" | "ppt";

interface VDoc {
  docId: string;
  docType: DocType;
  schemaVersion: string;
  title?: string;

  locale?: string;
  themeId?: string;

  assets?: AssetRef[];
  dataSources?: DataSourceDef[];
  queries?: QueryDef[];
  filters?: FilterDef[];

  root: VNode;
}
```

## 5.2 Node（统一模型）

```ts
type LayoutMode = "flow" | "grid" | "absolute";

interface VNode {
  id: string;
  kind: string;       // "chart"|"container"|"text"|"image"|"table"|"section"|"slide"...
  name?: string;

  layout?: VLayout;
  style?: VStyle;
  data?: VDataBinding;
  props?: Record<string, any>;

  children?: VNode[];
}
```

### Layout（平铺）

```ts
interface VLayout {
  mode: LayoutMode;

  gx?: number; gy?: number; gw?: number; gh?: number;   // grid
  x?: number; y?: number; w?: number; h?: number;       // absolute
  r?: number; z?: number;

  lock?: boolean;
  group?: string;
}
```

### Style（tokens + override 平铺）

```ts
interface VStyle {
  tokenId?: string;

  bg?: string; fg?: string;
  borderW?: number; borderC?: string;
  radius?: number; shadow?: string;
  pad?: number | [number,number,number,number];
  mar?: number | [number,number,number,number];

  font?: string; fontSize?: number;
  bold?: boolean; italic?: boolean; underline?: boolean;
  align?: "left"|"center"|"right";
}
```

### DataBinding（统一）

```ts
interface VDataBinding {
  sourceId: string;
  queryId?: string;

  params?: Record<string, string|number|boolean|string[]>;
  filterRefs?: string[];
}
```

# 6. Chart DSL（LLM 友好平铺设计）

## 6.1 Chart 节点

```ts
interface ChartNode extends VNode {
  kind: "chart";
  props: ChartSpec;
}
```

## 6.2 ChartSpec（平铺，减少嵌套）

```ts
type ChartType =
  | "auto"|"line"|"bar"|"pie"|"scatter"|"radar"|"heatmap"
  | "kline"|"boxplot"|"sankey"|"graph"|"treemap"|"sunburst"
  | "parallel"|"funnel"|"gauge"|"calendar"|"custom";

interface ChartSpec {
  chartType: ChartType;

  titleText?: string;
  subtitleText?: string;

  // 核心：字段绑定表（LLM 增删改最稳定）
  bindings: FieldBinding[];

  // 常用开关（平铺）
  legendShow?: boolean;
  legendPos?: "top"|"right"|"bottom"|"left";
  tooltipShow?: boolean;
  gridShow?: boolean;

  xAxisShow?: boolean;
  xAxisTitle?: string;
  xAxisType?: "category"|"value"|"time"|"log";

  yAxisShow?: boolean;
  yAxisTitle?: string;
  yAxisType?: "value"|"log";

  // 快捷风格（平铺）
  themeRef?: string;
  paletteRef?: string;
  smooth?: boolean;
  stack?: boolean;
  area?: boolean;
  labelShow?: boolean;

  // 格式化（平铺）
  valueFormat?: string;
  timeFormat?: string;

  // 行为（平铺列表）
  actions?: ChartAction[];

  // 专家兜底
  optionPatch?: Record<string, any>;
}
```

## 6.3 FieldBinding（行表）

```ts
type BindingRole =
  | "x"|"y"|"series"|"color"|"size"|"label"
  | "category"|"value"
  | "node"|"linkSource"|"linkTarget"|"linkValue"
  | "geo"|"lat"|"lng"
  | "tooltip"|"facet";

interface FieldBinding {
  role: BindingRole;
  field: string;

  agg?: "sum"|"avg"|"min"|"max"|"count"|"distinctCount"|"p50"|"p95"|"p99";
  as?: string;

  sort?: "asc"|"desc";
  topK?: number;

  format?: string;
  timeGrain?: "minute"|"hour"|"day"|"week"|"month";
  bin?: number;

  unit?: "bytes"|"bps"|"ms"|"pct"|"count";
}
```

## 6.4 ChartAction（联动）

```ts
type ActionType = "filter"|"drill"|"highlight"|"call"|"navigate";

interface ChartAction {
  on: "click"|"hover";
  type: ActionType;

  targetFilterId?: string;
  map?: { fromRole: BindingRole; toParam: string };

  fnName?: string;
  payload?: Record<string, any>;
  url?: string;
}
```

# 7. Dashboard / Report / PPT DSL（详细）

## 7.1 Dashboard（Grid 容器）

### 节点结构建议

* root.kind="container", layout.mode="grid"
* children：chart/text/image/table 等

DashboardProps（平铺）：

```ts
interface DashboardProps {
  dashTitle?: string;
  gridCols?: number;
  rowH?: number;
  gap?: number;

  bgMode?: "solid"|"image";
  bgAssetId?: string;

  showFilterBar?: boolean;
}
```

### Dashboard 功能规格（验收标准）

* [D-1] 可从图表库拖入图表到网格，出现占位预览
* [D-2] 支持 resize（拖拽角点），冲突时自动挤压/换位
* [D-3] 全局 filterBar：新增/删除/排序 filter；绑定字段/参数；影响关联图表
* [D-4] 支持多选卡片批量操作：对齐/统一主题/统一间距
* [D-5] 预览模式：锁定布局，隐藏编辑手柄

# 8. 统一编辑内核（详细设计）

## 8.1 Selection（选择系统）

* 单选/多选/框选
* 主选中（primary）与多选集合
* hover 高亮
* 锁定节点不可选（默认）

## 8.2 Command（命令系统）

所有修改统一走命令（UI 与 AI 一致）。

最小完备命令集：

* InsertNode
* RemoveNode
* MoveNode
* UpdateProps
* UpdateLayout
* UpdateStyle
* Batch / Transaction
* Group / Ungroup
* ApplyTheme（scope：doc/container/selection）
* ApplyTemplate（dashboard/ppt/report）

命令输出：

* patches[]
* inversePatches[]
* sideEffects（reflow/requery/rerender）

## 8.3 Patch / History / Merge

* Patch：JSON Patch 子集（add/remove/replace/move）
* History：undo/redo
* Merge：拖拽/resize 连续 patch 合并（按时间阈值或手势结束）

## 8.4 Diff Preview（AI可信）

* AI 提交 commandPlan → 生成 diff summary
* UI 展示变更：

  * 哪些 node 改动
  * 改了哪些字段
  * 影响范围（全局主题/某章节/某页）
* 支持 Accept / Reject / Undo

# 9. 数据与过滤（详细规格）

## 9.1 DataSourceDef（平铺）

支持 static / remote / polling / cache / retry / schema。

字段建议：

* id, type
* url, method, headers, params
* pollingEnabled, pollingInterval
* cacheEnabled, cacheTtl
* retryEnabled, retryMax, retryInterval
* schemaFields：[{name,type,label,unit,aggAble}]

## 9.2 QueryDef（可选，但推荐）

将 query 单独注册，binding 引用 queryId，便于复用与调试。

## 9.3 FilterDef（全局通用）

支持 scope（global 或某 container/node）。

字段：

* filterId, type, title
* bindField / bindParam
* scope
* optionsSourceId / optionsField / optionsStatic
* defaultValue

# 10. 主题与模板（低门槛核心能力）

## 10.1 Tokens / Theme

* tokens：颜色/字体/间距/圆角/阴影
* theme：token 集合 + 图表 theme 映射
* palette：调色板（用于 chart paletteRef）

## 10.2 Template / Snippet（必须）

* Dashboard 模板（网络运维：告警总览/流量趋势/设备健康）
* Report 模板（周报/月报：关键指标、异常、建议）
* PPT 模板（汇报：概览、问题、行动项）
* Snippet（片段）：标题+双图+结论块，一键插入

验收：

* [T-1] 新建时可选择模板
* [T-2] 选中容器可一键应用模板（产生节点与布局）
* [T-3] Snippet 支持拖入报告章节或 PPT 页面

# 11. ChatBI ↔ Canvas 交互（详细设计）

## 11.1 Command Plan（AI 输出格式）

AI 不直接输出 DSL，而输出“命令计划”。

```json
{
  "intent": "update",
  "targets": ["node_123"],
  "commands": [
    {"type":"UpdateProps","nodeId":"node_123","props":{"chartType":"line","smooth":true}},
    {"type":"UpdateProps","nodeId":"node_123","props":{"labelShow":true}}
  ],
  "explain": "将图表改为折线并开启平滑与标签"
}
```

## 11.2 对话场景规格（必须覆盖）

* 生成：生成图表并插入当前容器
* 修改：改图表类型/字段绑定/主题/布局
* 结构：在报告某章节插入图表与总结
* 批量：全局主题切换/所有图表开标签
* 拖拽：对话生成图表 → 可拖到 dashboard/ppt/report

## 11.3 安全机制（必须）

* Preview diff → Accept 执行
* 自动写入 history → 可 Undo
* 记录审计日志（本地即可）：谁（AI/UI）何时改了什么

# 12. 性能设计（验收标准）

* [Perf-1] 拖拽/resize 期间帧率稳定（>=55fps 目标，>=45fps 底线）
* [Perf-2] Report 200+ blocks 不掉帧（虚拟化）
* [Perf-3] Dashboard 50 图表可滚动与交互（懒加载）
* [Perf-4] 请求取消与 debounce 生效（避免频繁闪烁）
* [Perf-5] 批量命令通过 Transaction 合并减少 re-render

# 14. 用户旅程（User Journey）

## 14.1 小白：对话生成 → 拖入 Dashboard → 一键主题

1. 在对话框输入“最近7天告警趋势”
2. 生成图表卡片出现在“生成区”
3. 拖到 Dashboard 网格
4. 点击“一键主题：暗色科技”
5. 自动应用全局过滤“时间范围：7天”

验收：5分钟内完成可交付大屏。

## 14.2 分析师：精细绑定字段 + 全局过滤联动

1. 打开图表 Inspector
2. 修改 bindings：x=time（day），y=alarm_count（sum）
3. 添加 filter：region，多选
4. 选择 region=East → 所有关联图表刷新

## 14.3 汇报者：从 Report 一键生成 PPT

（预留扩展：Report→PPT 结构映射，或通过模板生成）

# 15. 默认策略（低门槛的关键）

## 15.1 图表默认策略

* chartType=auto：基于 bindings 推断
* 默认主题：继承 doc theme
* 默认 bindings 推荐：

  * 若存在 time 字段 → role=x
  * 若存在 metric 数值字段 → role=y（sum）
  * 若存在 category → role=series 或 color
* 默认格式化：bytes/bps/ms/pct 自动识别 unit

## 15.2 Dashboard 默认策略

* 新插入图表默认 gw/gh 按类型推荐（折线宽，饼图方）
* 冲突处理优先：挤压到最近空位
* 默认 filterBar：timeRange（最近7天）

## 15.3 PPT 默认策略

* 新页默认版式：标题+双图+总结
* 插入图表自动吸附到网格位
* 默认对齐线开启

# 16. 风险与对策

1. 自研 UI 成本高 → tokens/组件先做最小闭环，优先 Editor 核心组件
2. LLM 生成不稳定 → DSL 平铺 + command plan + diff 预览
3. 性能瓶颈 → 拖拽 transform、虚拟化、懒加载、transaction 合并
4. DSL 扩展失控 → registry + validator + schema 规范
