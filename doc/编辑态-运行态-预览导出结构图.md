## 组件职责速记

- `App.tsx`：统一管理路由、详情加载、动态预览快照、导出任务、编辑会话。
- `DetailPage.tsx`：详情页外壳，承接变量面板、最近导出信息和运行态内容。
- `DocRuntimeView.tsx`：按 `docType` 分发到 dashboard、report、ppt 三种运行态组件。
- `DashboardEditor.tsx / ReportEditor.tsx / PptEditor.tsx`：三类文档的编辑入口。
- `useDataEngine + useNodeRows + useNodeDataPrefetch`：编辑态与运行态共用的数据执行链。

## 这次问题对应到图里的位置

- 详情首屏不出数据：发生在 `DocRuntimeView -> *RuntimeView -> useDataEngine / useNodeRows / DataEngine` 这一段。
- PPT 查看布局错乱：发生在 `DocRuntimeView -> PptRuntimeView` 这一段。

## 当前修复结论

- 运行态三处 `*RuntimeView` 已去掉首屏请求上的额外防抖，避免详情页首开时当前可见节点请求被反复重置。
- `PptRuntimeView` 已显式保持节点绝对定位，避免运行态节点被通用样式覆盖后变成流式布局。
