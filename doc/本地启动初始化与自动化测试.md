# 本地启动初始化与自动化测试

本文档描述当前工程的统一启动、展示和全链路测试方式。目标是：

- 新环境一键初始化
- 本地一键拉起前后端联调
- 一键进入 showcase 展示模式
- 一键执行前端 + 后端 + 浏览器 smoke 的全流程测试

## 1. 前置要求

- Node.js 20+
- npm 10+
- JDK 21
- Maven 3.9+

默认端口：

- 前端开发：`5173`
- 前端 showcase：`4173`
- 后端 API：`18080`

## 2. 统一脚本入口

工程统一使用：

`node scripts/stack.mjs <command>`

也可直接使用 `package.json` 里的别名：

- `npm run stack:init`
- `npm run stack:dev`
- `npm run stack:showcase`
- `npm run stack:stop`
- `npm run test:all`

## 3. 初始化

首次在新环境执行：

```bash
npm run stack:init
```

该命令会完成：

- 安装前端 npm 依赖
- 编译后端测试类与依赖
- 安装 Playwright Chromium 浏览器

## 4. 本地联调

开发态联调：

```bash
npm run stack:dev
```

行为：

- 启动 `chatbi-app-server`
- 启动 Vite dev server
- 等待 `/api/v1/health` 和前端页面就绪
- 输出前后端访问地址

访问入口：

- 前端：`http://127.0.0.1:5173/#/docs`
- 后端健康检查：`http://127.0.0.1:18080/api/v1/health`

停止：

```bash
npm run stack:stop
```

日志位置：

- `.runtime/logs/backend-dev.log`
- `.runtime/logs/frontend-dev.log`

## 5. Showcase 演示

展示模式：

```bash
npm run stack:showcase
```

行为：

- 构建前端产物
- 清空 `.runtime/storage/showcase`
- 用干净种子数据启动后端
- 启动 `vite preview`

访问入口：

- `http://127.0.0.1:4173/#/docs`

适合：

- 方案演示
- 产品走查
- 稳定 UI 截图/录屏

日志位置：

- `.runtime/logs/backend-showcase.log`
- `.runtime/logs/frontend-showcase.log`

## 6. 自动化测试

### 6.1 单项测试

前端单测：

```bash
npm run test
```

前端类型检查：

```bash
npm run typecheck
```

后端集成测试：

```bash
npm run test:backend
```

浏览器 e2e：

```bash
npm run test:e2e
```

说明：

- `test:e2e` 需要先有可访问的前后端栈
- 默认读取 `CHATBI_E2E_BASE_URL`
- 未设置时默认访问 `http://127.0.0.1:4173`

### 6.2 全流程测试

一键跑完整链路：

```bash
npm run test:all
```

该命令会按顺序执行：

1. `stack:init`
2. 后端集成测试
3. 前端类型检查
4. 前端单元测试
5. 启动 showcase 栈
6. 执行 Playwright 全链路 smoke
7. 自动停止栈

## 7. 当前 smoke 用例覆盖

文件：

- `e2e/fullstack-smoke.spec.ts`

覆盖主链：

1. 前端文档中心从后端加载种子模板
2. 前端打开“数据接口管理”，读取后端 built-in endpoints
3. 前端为 dashboard 创建定时任务并立即执行
4. 前端在 report 详情页触发导出并拿到 artifact 链接
5. 前端进入 PPT 编辑器并发布当前改动

这套 smoke 目标不是覆盖所有细节，而是验证：

- 前端 UI
- Vite 代理/preview
- Java App REST
- SQLite 存储
- 调度执行链
- 导出执行链

已经串通。

## 8. 存储与运行目录

运行时目录：

- `.runtime/storage/dev`
- `.runtime/storage/showcase`
- `.runtime/logs`

其中：

- `dev` 用于本地调试
- `showcase` 用于演示和 e2e

## 9. 故障排查

后端没起来时先看：

- `.runtime/logs/backend-dev.log`
- `.runtime/logs/backend-showcase.log`

前端没起来时看：

- `.runtime/logs/frontend-dev.log`
- `.runtime/logs/frontend-showcase.log`

若端口被占用：

1. 执行 `npm run stack:stop`
2. 重新执行 `npm run stack:dev` 或 `npm run stack:showcase`

如果 Playwright 浏览器未安装：

```bash
npm exec playwright install chromium
```
