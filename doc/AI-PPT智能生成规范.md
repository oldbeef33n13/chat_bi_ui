# AI PPT 智能生成规范

## 1. 目标

PPT 智能生成的目标，是帮助用户快速得到一份适合汇报的 deck，而不是直接生成一堆不可控 slide。

核心原则：

`先页纲，再页面骨架，再页面内容，最后生成讲稿`

---

## 2. 使用场景

适用场景：

- 一句话生成汇报 PPT
- 从 report 转 PPT
- 对当前 slide 做优化
- 自动补下一页
- 生成 speaker notes

---

## 3. 生成阶段

## 3.1 阶段一：页纲生成

目标：

- 先确定整套 deck 的页结构

输出内容：

- 总页数建议
- 每页标题
- 每页目的

示例：

```json
{
  "deckTitle": "网络运营周汇报",
  "slides": [
    { "title": "本周总体结论", "goal": "先给领导结论" },
    { "title": "告警趋势与异常", "goal": "说明异常波动" },
    { "title": "容量风险", "goal": "说明链路压力" },
    { "title": "工单与处置效率", "goal": "说明执行效果" },
    { "title": "风险与建议", "goal": "形成行动项" }
  ]
}
```

## 3.2 阶段二：页面骨架生成

目标：

- 给每一页分配页面类型和布局模板

常见页面类型：

- 封面页
- KPI 页
- 趋势页
- 对比页
- 总结页
- 风险建议页

输出内容：

- 页面类型
- 推荐布局模板
- 建议组件

## 3.3 阶段三：页面内容生成

目标：

- 把具体内容填充到 slide

可生成：

- 标题
- 图表
- 关键结论
- bullet 文本
- 说明备注

## 3.4 阶段四：讲稿生成

目标：

- 帮用户形成汇报语言

可生成：

- speaker notes
- 一页一句话讲法
- 页间过渡

---

## 4. AI 可控制范围

AI 可以直接控制：

- slide 大纲
- 每页标题
- 页面类型
- 推荐布局
- 页面图表与文本组合
- 讲稿文本

AI 不建议直接控制：

- 大量像素级绝对定位
- 用户手工摆好的复杂细节位置
- 一次性重写全套 deck 的所有页面

---

## 5. 用户确认点

## 5.1 确认点一：页纲

用户确认：

- 一共有几页
- 每页讲什么

## 5.2 确认点二：页面骨架

用户确认：

- 每页类型是否合适
- 版式是否合适

## 5.3 确认点三：讲稿

用户确认：

- 语言风格是否符合汇报对象

---

## 6. PPT 智能生成 API 建议

建议拆成：

- `ppt_outline_generate`
- `ppt_slide_plan`
- `ppt_slide_generate`
- `ppt_notes_generate`
- `report_to_ppt_plan`

---

## 7. 当前结论

PPT 生成不应该直接一步到位生成所有页面细节。  
更合理的顺序是：

`页纲 -> 页面骨架 -> 页面内容 -> 讲稿`
