# AI Dashboard 智能生成规范

## 1. 目标

Dashboard 智能生成的目标，是帮助用户快速搭出“信息架构正确”的大盘，而不是直接做复杂细节排版。

核心原则：

`先模块结构，再组件计划，再布局，再洞察补充`

---

## 2. 使用场景

适用场景：

- 一句话生成大屏
- 一句话生成 PC 工作台
- 从已有模板派生新版本
- 自动补齐缺失模块
- 自动整理布局

---

## 3. 生成阶段

## 3.1 阶段一：模块结构生成

目标：

- 先决定整页信息架构

典型模块：

- KPI 汇总行
- 趋势区
- 对比区
- 明细表区
- 风险区
- 说明区

输出示例：

```json
{
  "displayMode": "fit_screen",
  "regions": [
    { "name": "kpi", "goal": "概览关键指标" },
    { "name": "trend", "goal": "看趋势变化" },
    { "name": "comparison", "goal": "看区域/对象对比" },
    { "name": "detail", "goal": "看明细和异常" }
  ]
}
```

## 3.2 阶段二：组件计划生成

目标：

- 为每个模块分配具体组件

可生成内容：

- KPI 卡片
- 图表
- 表格
- 文本说明
- 风险提示卡片

## 3.3 阶段三：布局草稿生成

目标：

- 给出高层布局建议

可控制内容：

- 组件大小级别
- 区域顺序
- 主次关系
- 是否适合全屏适配
- 是否适合页面滚动

不建议直接控制过细坐标。

## 3.4 阶段四：洞察补充

目标：

- 在已有 dashboard 上补即时结论和异常说明

可生成：

- 一句话洞察
- 风险提示
- 建议补充图表

---

## 4. AI 可控制范围

AI 可以直接控制：

- 模块结构
- 组件类型
- 数据绑定建议
- 高层布局建议
- 页面主题方向
- 洞察与说明

AI 不建议直接控制：

- 大量像素级位置细节
- 用户手工完成的复杂自由排布
- 整页不可逆大改

---

## 5. 用户确认点

## 5.1 确认点一：页面结构

用户确认：

- 要不要 KPI 行
- 要不要趋势区
- 要不要明细区

## 5.2 确认点二：组件计划

用户确认：

- 每个区域放什么图
- 数据是否合适

## 5.3 确认点三：布局整理

用户确认：

- 是否自动整理页面
- 是否保留现有布局

---

## 6. Dashboard 智能生成 API 建议

建议拆成：

- `dashboard_outline_generate`
- `dashboard_block_plan`
- `dashboard_layout_suggest`
- `dashboard_insight_generate`

---

## 7. 当前结论

Dashboard 最适合 AI 的，不是细碎的自由布局控制，而是：

- 帮你先搭对信息架构
- 帮你补齐关键模块
- 帮你生成即时洞察
