## 附件1：完整 JSON Schema（VDoc / Node / ChartSpec / Filter / DataSource / Query / CommandPlan）

> 说明
>
> * 采用 **Draft 2020-12**
> * Schema 结构尽量 **平铺**，减少深层嵌套（尤其 ChartSpec）
> * `VNode.props` 在不同 kind 下通过 `oneOf` 约束（chart/container/section/slide/text 等）
> * `ChartSpec.bindings` 使用“行表”设计，LLM 增删改最稳
> * `CommandPlan` 是 Chat ↔ Canvas 的契约：AI 只输出命令计划，不直接写 DSL

## 附件2：交互细则与验收用例表（覆盖核心能力 + 边界 + 性能）

> 说明
>
> * 用例按模块分组：Core Editor Kernel / Chart / Dashboard / Report / PPT / Data & Filter / Chat Bridge / Theme & Template / Perf & Reliability
> * 每条用例都给：前置条件、步骤、预期结果、边界/异常、性能指标
> * 你可以直接交给测试或作为验收标准

### A. Core Editor Kernel（Selection / Command / Patch / History）

| 用例ID | 场景             | 前置条件            | 操作步骤             | 预期结果                             | 边界/异常              | 性能/指标           |
| ---- | -------------- | --------------- | ---------------- | -------------------------------- | ------------------ | --------------- |
| K-01 | 单选/多选          | 画布有多个节点         | 点击节点A；按Ctrl点击节点B | selectedIds=[A,B]，primary=B；高亮正确 | 锁定节点不可被选中（或提示解锁）   | 选中反馈<50ms       |
| K-02 | 框选             | PPT或Dashboard画布 | 鼠标拖拽框选覆盖多个元素     | 选中被框住元素集合                        | 框选跨容器时只选当前容器       | 55fps 目标        |
| K-03 | 右键快捷工具条        | 任意节点可选中         | 选中节点，右键打开工具条     | 展示高频功能（复制/删除/对齐等）                | 空白处右键展示画布菜单        | 弹出<80ms         |
| K-04 | Undo/Redo 基础   | 执行过一个更新         | Ctrl+Z；Ctrl+Y    | 文档状态回退/重做一致                      | 连续拖拽操作应合并为一次       | Undo/Redo<100ms |

| K-06 | Patch 最小化      | 开启调试            | 更新某节点某字段         | patch 仅包含对应 path replace         | patch path 需稳定、可重放 | patch生成<30ms    |
