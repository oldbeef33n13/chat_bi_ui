# 代码注释与扩展指南（前端）

本文档配合源码中的中文注释，帮助后续按模块扩展，而不是“到处找入口”。

## 1. 文档中心与仓储层
- 入口: `src/ui/hooks/use-doc-library.ts`
- 接口契约: `src/ui/api/doc-repository.ts`
- HTTP 实现: `src/ui/api/http-doc-repository.ts`
- 本地兜底: `src/ui/api/local-doc-repository.ts`

扩展建议:
- 新增后端字段时，优先改 `doc-repository.ts` 类型，再改 http/local 两个实现。
- 需要灰度策略时，保持 `DocRepository` 不变，在 `use-doc-library` 做路由/开关。

## 2. 编辑器状态内核
- 核心: `src/core/kernel/editor-store.ts`
- 上下文桥接: `src/ui/state/editor-context.tsx`

扩展建议:
- 一切可撤销操作都走 `executeCommand/executeCommands`。
- AI 修改走 `previewPlan -> acceptPreview/rejectPreview`，不要直接“静默改 doc”。

## 3. 三类编辑器
- Dashboard: `src/ui/editors/DashboardEditor.tsx`
- Report: `src/ui/editors/ReportEditor.tsx`
- PPT: `src/ui/editors/PptEditor.tsx`
- 编辑器壳与懒加载: `src/ui/components/CanvasPanel.tsx`

扩展建议:
- 新交互先在各编辑器实现，再通过 `CanvasPanel` 做统一入口。
- 若新增第四种编辑器，先补 `DocType` + `CanvasPanel` 懒加载分块。

## 4. 图表推荐与 AI 交互
- 推荐引擎: `src/ui/utils/chart-recommend.ts`
- 图表追问: `src/ui/utils/chart-assistant.ts`
- 图表追问 UI: `src/ui/components/ChartAskAssistant.tsx`
- Chat 驱动改图: `src/ui/components/ChatBridgePanel.tsx`

扩展建议:
- 接大模型时只需注册 `registerChartRecommendProvider`。
- 复杂问答可替换 `askChartAssistant`，保持输出结构不变即可复用 UI。

## 5. AI 埋点
- 统一埋点层: `src/ui/telemetry/ai-telemetry.ts`
- 全局上下文注入: `src/ui/App.tsx`

扩展建议:
- 生产环境接入时，注册 `registerAiTelemetrySink` 并上送后端。
- 事件字段尽量保持兼容，新增字段放 `meta`，避免破坏消费端。

## 6. 关键算法工具
- Dashboard 冲突解算: `src/ui/utils/dashboard-grid.ts`
- Report 虚拟化: `src/ui/utils/report-virtual.ts`

扩展建议:
- 算法优化时优先补测试（同目录 `*.test.ts`），再调实现。
